# Slack Rules

## Sending messages

Never send a Slack message without showing the user a draft first. Always confirm: channel/DM, message content, and whether it should be threaded.

## Tone

- Match the channel's tone — casual in #random, professional in #engineering
- Keep messages concise — Slack is not email
- Use threads for follow-ups, not new messages in the channel

## Mentions

Never @mention someone unless the user explicitly asks. Unwanted pings are disruptive.

## Scheduling

If the message is not urgent and it's outside working hours, suggest scheduling it for the next business day morning instead of sending immediately.
