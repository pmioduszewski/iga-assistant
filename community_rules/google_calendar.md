# Google Calendar Rules

Aliases: gmail, gcal

## Before creating events

Always list existing events for the target day before creating a new one. Check for time conflicts and duplicates. Never create an event without verifying the slot is free.

## Before modifying events

Fetch the event first. Show the user what will change. Never modify without confirmation.

## Defaults

- Default duration: 30 minutes (unless the user specifies otherwise)
- Default reminder: 10 minutes before
- If the user says "meeting" without specifying duration, use 60 minutes

## Color coding

Define your own color scheme here. Example:

- Color 6 (Orange) = Work
- Color 2 (Sage) = Personal / health
- Color 7 (Peacock) = Side project
- Color 3 (Grape) = Family

Edit the colors above to match your workflow. Gaia will apply them when creating events.

## Recurring events

When creating recurring events, always confirm the recurrence pattern with the user before saving. Show: start date, end date (or "no end"), frequency, exceptions.
