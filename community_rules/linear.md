# Linear Rules

## Issue creation

Draft issues in chat before creating them in Linear. Show: title, description, team, priority, labels. Wait for user approval.

## Writing style

- Concise titles — under 80 characters
- Descriptions: context first, then acceptance criteria
- No emojis in titles
- Use markdown formatting in descriptions

## Priority mapping

- Urgent = blocking deployment or breaking production
- High = needed for current cycle
- Medium = planned work, current or next cycle
- Low = backlog, nice to have
- No priority = triage needed

## Labels and projects

Ask the user about their label and project conventions on first use. Store preferences in MemPalace `gaia/rules` wing for future reference.

## Cycle awareness

When creating issues, check the current active cycle and suggest adding the issue to it if relevant. Don't add to a cycle without asking.
