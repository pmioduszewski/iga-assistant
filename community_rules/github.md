# GitHub Rules

## Pull requests

- Keep PR titles under 70 characters
- PR description: summary (2-3 bullets), test plan, any migration notes
- Always check CI status before asking for review
- Link related issues in the description

## Issues

Draft issues in chat before creating. Show: title, body, labels, assignees. Wait for approval.

## Code review

When reviewing PRs, focus on:
1. Correctness — does it do what it claims?
2. Security — any obvious vulnerabilities?
3. Maintainability — will future readers understand this?

Skip style comments unless they affect readability. Don't suggest changes that are purely preference.

## Commits

- Conventional commit style preferred (feat:, fix:, docs:, refactor:, etc.)
- Subject line under 50 characters
- Body explains why, not what
