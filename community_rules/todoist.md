# Todoist Rules

## Task structure

Always use subtasks for multi-step work. Never use description checkboxes — they are not searchable and not visible in most views.

## Priorities

- P1 = urgent and important (deadline today or blocking others)
- P2 = important but not urgent
- P3 = nice to have
- P4 = someday / low priority

Default to P4 unless the user specifies otherwise or the context clearly implies urgency.

## Due dates

Use natural language due dates ("tomorrow", "next Monday") rather than ISO dates. Todoist handles them well and they're easier to read in chat.

## Shared lists

If a list is shared with someone, always write task titles in the language the other person expects. Ask on first use if unsure.

## Projects

When creating tasks, always assign them to a project. If unsure which project, ask. Never leave tasks in the inbox without a project.
