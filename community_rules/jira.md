# Jira Rules

Aliases: atlassian

## Issue creation

Never create Jira issues directly. Always draft the issue in chat first — show title, description, type, priority, and assignee. Wait for user approval before creating.

## Issue content

- Write in the project's working language (ask user which language on first use)
- Concise, human-readable style — no AI essays, no bullet-point walls
- No emojis in issue titles or descriptions
- Keep descriptions under 200 words unless the issue genuinely requires more

## Watermark

When Iga creates or updates a Jira issue, add a small attribution at the bottom of the description:

> ###### *Created with Iga — Personal AI Assistant*

## Status transitions

Before transitioning an issue, show the user the current status and the target status. Never transition without confirmation.

## Comments

When adding comments to issues, keep them factual and brief. No "Great progress!" filler.
